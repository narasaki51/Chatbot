# 🗼 Chzzk Ladder Game & Mission Board Technical Guide

이 문서는 치지직 방송용 **'의리사다리 게임'**의 정밀한 UI/UX 규격과 로직을 보호하기 위한 가이드입니다. 향후 다른 AI나 개발자가 코드를 수정할 때, **이 문서의 규칙을 반드시 준수**해야 합니다.

## 1. 핵심 디자인 규격 (CRITICAL DESIGN RULES)

> [!IMPORTANT]
> 아래 수치들은 수많은 시행착오 끝에 방송 화면(OBS/Overlay)에 최적화된 값이므로, **사용자의 명시적 요청 없이 수정하지 마십시오.**

*   **전체 너비 (WIDTH):** `1000px` 고정 (컴팩트 와이드 바 스타일).
*   **사다리 디자인:** 
    *   세로 기둥(Pole) 및 가로 다리(Rung) 굵기: **4px** (슬림 스타일).
    *   진행되는 활성 선 굵기: **8px**.
    *   색상 컨셉: **#ffbd2e (앤틱 골드)** 메인, **#00ffa3 (에메랄드)** 포인트.
*   **입력창 정렬 (PIXEL-PERFECT SYNC):** 
    *   **방식:** SVG 내부의 `foreignObject`를 사용하여 사다리 기둥과 입력창을 결합.
    *   **정렬 공식:** `x = (i / (count - 1)) * WIDTH - (박스너비 / 2)`.
    *   **절대 주의:** 입력 박스는 반드시 사다리 줄 정중앙에 수직으로 일치해야 하며, 어떠한 경우에도 좌우로 어긋나서는 안 됩니다.
*   **박제된 입력창 크기:** 이름 입력창(~130px), 결과 입력창(~160px)으로 기존보다 2배 확장된 상태를 유지.

## 2. 핵심 로직 가이드 (LOGIC & STABILITY)

### ⚠️ 화면 전환 블랙아웃(Crash) 방지
*   미션 카드의 **[사다리 가기]** 버튼 클릭 시, 반드시 `setView('ladder')`를 먼저 실행하여 **화면을 먼저 전환한 뒤** 미션 데이터를 삭제(`updateStatus`)해야 합니다. (동시 실행 시 렌더링 충돌로 화면이 멈출 수 있음)

### ✨ 역동적인 미션 연출
*   성공/실패 시 `framer-motion`을 이용한 약 0.8초의 **애니메이션 연출**이 먼저 실행된 후, 서버에 데이터 삭제 요청을 보내야 합니다.
    *   **Success:** Scale up & Green glow.
    *   **Failure:** Horizontal Shake & Red burn.

### 🙈 블라인드 모드 정체성
*   블라인드 모드 활성 시 가로 다리와 진행 경로의 `opacity`를 0으로 처리하며, 이 과정에서 **레이아웃 흔들림이 없어야 함**을 보충해야 합니다.

## 3. 기술 스택 및 연동
*   **Server:** Node.js (Express), Socket.io.
*   **Database:** 프로젝트 루트의 `missions_db.json` (영구 저장).
*   **Frontend:** React, Framer Motion, Lucide Icons.

## 4. 향후 수정 시 주의사항
*   **인원수 변경:** 사다리 인원수는 최소 2명, 최대 10명으로 제한되어 있으며, 이에 따라 사다리판이 유동적으로 반응하지만 **1,000px 너비**는 유지되어야 합니다.
*   **ViewBox Margin:** 가장자리 입력창이 잘리지 않도록 SVG `viewBox`의 좌항 여백(`-100` 이상)이 반드시 확보되어 있어야 합니다.
